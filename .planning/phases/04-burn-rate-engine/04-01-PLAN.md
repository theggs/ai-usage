Create `src/lib/tauri/burnRate.ts` with these exported types and functions:
```typescript
import type { BurnRateSample } from "./contracts";

export type BurnRatePace = "on-track" | "behind" | "far-behind";

export interface BurnRateDisplay {
  pace: BurnRatePace;
  depletionEtaMs: number | null;
  willLastUntilReset: boolean;
}

export const getBurnRateDisplay = ({
  remainingPercent,
  resetsAt,
  samples,
  nowMs
}: {
  remainingPercent?: number;
  resetsAt?: string;
  samples?: readonly BurnRateSample[];
  nowMs?: number;
}): BurnRateDisplay | undefined => { ... }
```

The implementation must follow the locked model exactly:
- Per D-07, use sample-based ETA-vs-reset projection. Do not infer elapsed-window budget from labels and do not add `windowDurationMinutes`.
- Parse `resetsAt` with `Date.parse`. If it is absent, unparseable, or `<= nowMs`, return `undefined` per D-12 and D-13.
- Filter samples to valid ISO/epoch timestamps, sort ascending by timestamp, and require at least 2 valid samples.
- Use the oldest and newest valid samples only.
- Compute `consumed = first.remainingPercent - last.remainingPercent` and `elapsedMs = lastMs - firstMs`.
- If `elapsedMs <= 0`, return `undefined`.
- If `consumed <= 0`, return `{ pace: "on-track", depletionEtaMs: null, willLastUntilReset: true }`.
- Compute `ratePerMs = consumed / elapsedMs`, `depletionEtaMs = remainingPercent / ratePerMs`, `timeUntilResetMs = resetMs - nowMs`, and `coverage = depletionEtaMs / timeUntilResetMs`.
- Use deterministic thresholds per D-04 and D-05:
  - `coverage >= 1` => `"on-track"`
  - `coverage >= 0.5 && coverage < 1` => `"behind"`
  - `coverage < 0.5` => `"far-behind"`

Update `src/lib/tauri/summary.ts` without changing existing alert colors:
- keep `getQuotaStatus` and `getQuotaProgressTone` untouched
- export `getBurnRateDisplay` from `./burnRate`
- if needed, add a small wrapper `getQuotaBurnRateDisplay(dimension: QuotaDimension, nowMs?: number)` that passes `dimension.remainingPercent`, `dimension.resetsAt`, and `dimension.burnRateHistory`

Update `src/lib/tauri/summary.test.ts` with explicit coverage for the 8 behaviors above. Keep tests in Vitest; do not add new libraries.
