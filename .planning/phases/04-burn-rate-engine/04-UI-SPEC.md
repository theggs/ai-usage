| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Quota row gaps, card internal grouping, compact vertical rhythm |
| md | 16px | Default card padding, progress-track height, standard content spacing |
| lg | 24px | Section padding in onboarding/disclosure blocks, larger shell groups |
| xl | 32px | Large layout separation inside full panel stacks |
| 2xl | 48px | Major section breaks |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Heading | 16px (`text-base`) | 600 | 1.5 |
| Metric | 14px (`text-sm`) | 600 | 1.3 |
| Label | 12px (`text-xs`) | 600 | 1.25 |
| Supporting | 12px (`text-xs`) | 400 | 1.4 |

**Rules for Phase 4:**
- Burn-rate pace label uses the `Label` role, sentence case, never uppercase.
- Burn-rate ETA / positive confirmation uses the `Supporting` role.
- Burn-rate copy must stay within exactly two text lines total: one pace line and one secondary line.
- Existing service title, remaining value, severity pill, and reset hint sizes stay unchanged.

**Source:** Inherited from `src/components/panel/QuotaSummary.tsx`, `src/components/panel/ServiceCard.tsx`, and existing bilingual panel copy patterns in `src/app/shared/i18n.ts`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | App background, service cards, primary surfaces |
| Secondary (30%) | `#f8fafc` (`slate-50`) | Quota dimension containers and muted sub-surfaces |
| Accent (10%) | `#10b981` (`emerald-500`) | Existing healthy progress fills and enabled-state controls only |
| Destructive | `#f43f5e` (`rose-500`) | Existing danger-state bars, borders, and badges only; no new destructive action in this phase |

Accent reserved for: existing healthy progress bars, enabled toggle tracks, and established positive-state pills only. Phase 4 must not use accent, amber, or rose as the primary differentiator for burn-rate labels.

**Burn-rate-specific color contract:**
- Pace label text: `text-slate-700` for all three pace classes.
- Secondary ETA / confirmation line: `text-slate-500`.
- No new badge, chip, icon, border stripe, or background tint is introduced for burn-rate output.
- Existing percentage progress color logic remains the single source of red/amber/green severity in Phase 4.

**Reason:** Phase boundary explicitly excludes warning-color logic changes; burn-rate communicates through wording, not a new severity palette.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Inherit existing panel CTA: `Refresh` / `手动刷新` |
| Empty state heading | none introduced |
| Empty state body | Burn-rate output is omitted entirely when history has fewer than 2 successful samples or `resetsAt` is missing/invalid. Keep the existing quota row UI without placeholder text. |
| Error state | No burn-rate-specific error sentence. Reuse existing `SnapshotStatus` cards and placeholder messaging; burn-rate itself fails silent. |
| Destructive confirmation | none |

### Pace Labels

| State | en-US | zh-CN |
|-------|-------|-------|
| On track | On track | 进度正常 |
| Behind | Behind | 消耗偏快 |
| Far behind | Far behind | 消耗过快 |

### Secondary Line

| Condition | en-US | zh-CN |
|-----------|-------|-------|
| Projected to last | Will last until reset | 可撑到重置 |
| Projected to deplete early | Runs out in ~{value} | 约 {value} 后用尽 |

### Copy Rules

- Use sentence case, no trailing period.
- Keep wording short enough to fit a narrow menu-bar panel without wrapping beyond the prescribed two lines.
- Never show speculative copy such as `Calculating...`, `Estimating...`, or `No prediction yet`.
- Do not introduce provider-specific burn-rate wording; the same copy applies to every quota dimension.

## Interaction Patterns

### Placement

- Burn-rate output appears inside each quota dimension block, below the progress bar and above the existing reset hint.
- Layout order stays: header row -> progress bar -> burn-rate line 1 -> burn-rate line 2 -> reset hint.
- Burn-rate text is left-aligned. Existing reset hint remains the lowest-priority line and stays right-aligned.

### Visibility Rules

- Show burn-rate output only when both conditions are true:
  - at least 2 successful persisted samples exist for that provider + dimension key
  - `resetsAt` parses to a future timestamp judged valid by the implementation
- If either condition fails, render nothing for burn-rate. Do not reserve empty space for a missing block.
- Missing burn-rate data must never affect the rendering of remaining percentage, progress bar, reset hint, or existing service severity styling.

### Update Behavior

- Burn-rate content is read-only. There is no tap target, tooltip, hover affordance, or disclosure.
- History capture happens only on successful provider refresh.
- While the panel is visible, ETA / confirmation text may recompute on the existing minute ticker, but that ticker must not create synthetic history.
- Text updates should be immediate and unanimated; keep the existing width transition on the progress bar only.

### Visual Constraints

- Pace state is distinguished by wording only, not by a new color chip or icon.
- The burn-rate block must remain visually subordinate to the remaining-percentage metric.
- If horizontal space becomes tight in Chinese or long provider labels, shorten copy rather than allowing a third burn-rate line.
