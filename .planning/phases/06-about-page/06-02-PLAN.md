## Locked decisions to implement
- D-12: Show dependency total plus copyleft status in the About page.
- D-13: Audit must cover both `Cargo.lock` and `package-lock.json`.
- D-14: Audit output is a generated JSON file bundled via static import.

## Review concerns to address
- HIGH: current Rust audit path sets every crate license to `null`, so ABOUT-05 is not actually met.
- MEDIUM: npm resolution silently misses nested or scoped packages.
- MEDIUM: missing-license packages are currently treated as permissive instead of being surfaced as unknown.

## Chosen implementation direction
- npm source of truth: `package-lock.json` `packages` map keyed by full install path (lockfile v3), with fallback to the matching `package.json` on disk only when the lock entry lacks a license field.
- Rust source of truth: `src-tauri/Cargo.lock` plus local crate manifests under `$CARGO_HOME/registry/src/*/{name}-{version}/Cargo.toml` (and `$CARGO_HOME/git/checkouts` for git crates when present). Do not depend on network lookups.
- Failure guard: if an ecosystem has packages but resolves zero licenses for that ecosystem, exit non-zero so the build cannot ship a fake green audit.
</context>
