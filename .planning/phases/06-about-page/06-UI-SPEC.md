## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Inline gaps between icon and text |
| sm | 8px | Compact element spacing, gap between key-value rows |
| md | 16px | Default element spacing, px-4 page padding (matches AppShell) |
| lg | 24px | Hero section vertical spacing between icon/name/version |
| xl | 32px | Gap between hero section and key-value list |
| 2xl | 48px | Top padding for hero section (visual breathing room) |

Exceptions:

| Value | Tailwind | Justification |
|-------|----------|---------------|
| 14px | `py-3.5` | Matches existing SettingsView `rowClassName` pattern (`const rowClassName = "px-5 py-3.5"` at SettingsView.tsx:53). Used across 10+ rows in SettingsView. Key-value card rows in About must use the same padding for visual consistency between views. |
| 2px | `py-0.5` | Inline micro-spacing for copyleft warning badge (`text-[11px]` with `rounded-full`). Standard Tailwind pattern for tight pill badges where `py-1` (4px) would make the badge disproportionately tall relative to 11px text. |

Source: Existing AppShell.tsx and SettingsView.tsx spacing patterns.

## Copywriting Contract

All copy is delivered through `i18n.ts` using the existing `getCopy()` pattern with zh-CN and en-US locales.

| Element | Key | zh-CN | en-US |
|---------|-----|-------|-------|
| About page title (header) | aboutTitle | "About" | "About" |
| Settings footer link | aboutLink | "About AIUsage >" | "About AIUsage >" |
| License row label | aboutLicenseLabel | "License" | "License" |
| License row value | (static) | "Apache 2.0" | "Apache 2.0" |
| GitHub row label | aboutGitHubLabel | "GitHub" | "GitHub" |
| GitHub row value | (static) | "github.com/user/ai-usage" | "github.com/user/ai-usage" |
| Dependencies row label | aboutDependenciesLabel | "Dependencies" | "Dependencies" |
| Dep audit: all permissive | aboutDepsAllPermissive | "{count} packages / All permissive" | "{count} packages / All permissive" |
| Dep audit: copyleft found | aboutDepsCopyleftFound | "{count} packages / {copyleft} copyleft" | "{count} packages / {copyleft} copyleft" |

**No empty state:** The About page always has content (version comes from Tauri API, license is static, dependency audit is build-time generated). There is no data-dependent empty state.

**No error state:** The About page displays static/build-time data. If `app.version` fails, fall back to the version string from `package.json` bundled at build time. No network requests, no error paths.

**No destructive actions:** The About page is read-only. No delete, reset, or irreversible operations.

**Primary CTA:** None. The About page is informational with no call to action. The GitHub link is the only interactive element.

Source: Decisions D-09, D-10, D-11, D-12 from CONTEXT.md.

## Interaction Contract

### Navigation Flow

```
Panel --[gear button]--> Settings --[footer link]--> About
About --[back arrow]--> Settings --[back arrow]--> Panel
```

- About is always accessed through Settings, never directly from Panel
- Window blur resets to Panel (existing behavior covers "about" since resetShellViewState sets currentView to "panel")
- No keyboard shortcut for About page (D-04)

### Slide Animation

Extend the existing 2-view slide to 3-view:

| View | Container Width | Translate X |
|------|----------------|-------------|
| Panel | w-[300%] | translate-x-0 |
| Settings | w-[300%] | -translate-x-[calc(33.333%+0.667rem)] |
| About | w-[300%] | -translate-x-[calc(66.667%+1.333rem)] |

Animation: `transition-transform duration-300 ease-out` (existing).

Gap between views: `gap-4` (existing, 16px).

Note: The translate offsets account for the gap. With 3 children each at 33.333% width and 2 gaps of 1rem (16px) between them, the second view starts at 33.333% + 1 gap, and the third at 66.667% + 2 gaps. The exact calc values should be verified during implementation against the actual rendered layout.

### GitHub Link Behavior

- Rendered as a `<button>` styled as a link (not an `<a>` tag) to avoid default browser navigation inside the Tauri webview
- On click: call `import { open } from "@tauri-apps/plugin-shell"; open(url)` to open in default browser
- Visual: text-blue-600, hover:text-blue-700, hover:underline, cursor-pointer
- Truncate the displayed URL if it exceeds the available width

### Scroll Behavior

- The About page content fits within the 792px window height without scrolling
- If future fields make content exceed viewport, the About view container should have `overflow-y-auto` (consistent with Settings)
- Header shadow on scroll: reuse existing `isScrolled` pattern from AppShell

## Build-Time Dependency Audit

### Audit Data Shape

The build script generates a JSON file at a known path that Vite imports statically:

```typescript
// src/generated/license-audit.json (generated at build time)
{
  "totalPackages": 87,
  "copyleftCount": 0,
  "generatedAt": "2026-04-02T12:00:00Z"
}
```

- Imported in AboutView via `import auditData from "../../generated/license-audit.json"`
- If the file does not exist at build time (dev mode), fall back to `{ totalPackages: 0, copyleftCount: 0 }`
- Display logic: if `copyleftCount === 0`, show "All permissive"; otherwise show "{copyleftCount} copyleft" in amber warning style

### Copyleft Warning Visual

When `copyleftCount > 0`:
- The dependency value text changes from text-slate-500 to text-amber-700
- A small inline badge appears: `rounded-full bg-amber-50 border border-amber-200 px-2 py-0.5 text-[11px] font-semibold text-amber-700` (see Spacing Scale exceptions for `py-0.5` justification)
- Badge text: "{count} copyleft"

When `copyleftCount === 0`:
- Show "All permissive" in text-slate-500 (no special treatment)
- Do NOT show "0 copyleft" -- only flag when copyleft is found (discretion area from CONTEXT.md)
