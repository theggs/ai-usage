## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Background, surfaces, card backgrounds |
| Secondary (30%) | slate-50 (`#f8fafc`) | Quota dimension backgrounds, input backgrounds, service pills |
| Accent (10%) | emerald-500 (`#10b981`) | Healthy progress bars, enabled toggle tracks |
| Destructive | rose-500 (`#f43f5e`) | Danger progress bars, danger card borders, error text |

**Semantic color mapping (progress bars and card states):**

| Condition | Progress bar | Card border | Card background | Text |
|-----------|-------------|-------------|-----------------|------|
| Healthy (>50%) | emerald-500 | slate-200 | white | emerald-700 |
| Warning (20-50%) | amber-400 | amber-200 | amber-50/70 | amber-700 |
| Danger (<20%) | rose-500 | rose-200 | rose-50/70 | rose-700 |
| Muted (no data) | slate-300 | slate-200 | white | slate-700 |

Accent reserved for: progress bars at healthy state, toggle switch tracks when enabled, active-window promotion pills. Never used for text links, buttons, or decorative elements.

**Source:** Existing ServiceCard.tsx, QuotaSummary.tsx `toneClasses` / `textToneClasses` maps. Decision D-09 mandates identical visual treatment.

### Error / Status States

No new error copy. All states reuse existing SnapshotStatus messages (per D-07):

| State | Existing copy key | When shown |
|-------|------------------|------------|
| No token configured | `serviceNotInstalledTitle` / `serviceNotInstalledBody` | Provider enabled but no token in preferences |
| API unreachable | `serviceDisconnectedTitle` / `serviceDisconnectedBody` | HTTP error or timeout |
| Rate limited | `statusRateLimitedTitle` / `statusRateLimitedBody` | 429 response |
| Auth failed | `statusAccessDeniedTitle` / `statusAccessDeniedBody` | 401/403 response |
| Provider disabled | (not shown in panel) | `providerEnabled[id] = false` |

### Empty State

No new empty state. When a new provider has no data yet, it renders as a ServiceCard with `SnapshotStatus::NoCredentials`, showing the existing "not connected" messaging with a prompt to add a token in Settings.

### Destructive Actions

None in this phase. Disabling a provider is a toggle (non-destructive, reversible). Removing a token is done by clearing the input field (no confirmation needed -- same pattern as proxy URL).

## Interaction Patterns

### Token Input

- Pattern: Identical to existing proxy URL input (`inputClassName`, `onBlur` commit, `onChange` draft)
- Input type: `password` (masks token value)
- Validation: Non-empty string. No format validation beyond presence (API call is the true validator)
- Commit trigger: `onBlur` (same as proxy URL)
- Error display: Via PreferenceField `error` prop if API returns auth failure after token save

### Provider Enable/Disable Toggle

- Pattern: Identical to existing Claude Code toggle in SettingsView
- Toggle appearance: `w-14` pill switch, emerald-500 when enabled, slate-300 when disabled
- State: `providerEnabled[provider_id]` in UserPreferences
- Conditional display: Token input field only visible when provider is enabled

### GLM Region Selector

- Pattern: Standard `<select>` with `selectClassName` (same as language/proxy mode selectors)
- Options: Global (z.ai) | China (bigmodel.cn)
- Commit: Immediate via `applyImmediatePatch` (same as other select fields)
- Storage: New field in UserPreferences (e.g., `glmRegion: "global" | "china"`)

### Service Order Integration

- New providers appear in the drag-reorder pill row automatically via registry
- No interaction changes needed -- existing drag logic handles N providers
