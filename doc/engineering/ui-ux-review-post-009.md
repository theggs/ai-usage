## 新发现的问题

### 25.【P1】面板底部卡片被截断，缺少底部间距
**截图实证**：面板截图中 Codex 卡片被窗口底部截断，最后一张卡片紧贴滚动容器边缘，无呼吸空间。
**问题**：用户滚动到最底部时，最后一张卡片紧贴容器底边，视觉上缺乏完成感，且容易误以为下方还有被遮挡的内容。
**建议**：在 PanelView 或滚动容器底部添加 `pb-4` 间距。

### 26.【P2】设置页分组标题与卡片间距过大
**截图实证**：设置截图中"通用"标题和第一张卡片（语言）之间、"显示"标题和前一张卡片（刷新间隔）之间，间距视觉上偏大。去掉 PreferenceSection 的卡片包裹后，组标题和内容之间缺少视觉归属感。
**问题**：去卡片化后，分组标题和下方内容的从属关系不够紧密。用户可能不确定标题是属于上方还是下方。
**建议**：
- 组标题上方间距大于下方间距（如上 `mt-5`、下 `mb-1`），拉近标题与其内容的视觉距离
- 或在组标题下加一条细分隔线

### 27.【P2】代理"应用"按钮视觉权重过高
**代码实证**：`SettingsView.tsx:269` 代理应用按钮使用 `bg-emerald-50 border-emerald-200 text-emerald-900 font-semibold`，是设置页中唯一带颜色填充的按钮。
**问题**：代理设置是低频操作，但其"应用"按钮的视觉权重高于所有即时生效的设置项，造成注意力吸引不合理。与 FR-019 对通知测试按钮的降权处理不一致。
**建议**：降为 `border-slate-200 bg-white text-slate-700`（与通知测试按钮同级），或仅在代理配置有未保存更改时才高亮。

### 28.【P2】面板与设置 header 布局不对称
**截图实证**：面板 header 左侧有两行信息（副标题+摘要），右侧两个按钮（刷新+设置）；设置 header 左侧仅一行小字"设置"，右侧"< 返回"按钮。
**问题**：两个 header 的信息密度和视觉重心差异大。设置 header 左侧空间利用率低，显得空旷，与面板 header 的紧凑感不匹配。
**建议**：设置 header 可以在"设置"文字下方加一行当前保存状态（如"已保存"/"保存中..."），利用空间同时提供即时反馈。
