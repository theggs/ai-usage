## 3. P0 — 必须解决

### 3.1 进度条内文字重复（#38）

**现状：** 进度条左侧显示 `47%`，右侧显示 `剩余 47%`，同一信息出现两次。
**目标：** 进度条内仅保留一条百分比文字（如 `剩余 47%`），居中或靠左显示。
**注意事项：** 文字需在进度条填充区（深色背景）和空白区（浅色背景）上都保持足够对比度。

### 3.2 手动/自动刷新后菜单栏数据不更新（#39）

**现状：** 面板刷新后，面板内显示最新数据，但菜单栏仍显示旧数据。仅在偏好设置变更时菜单栏才会同步。
**目标：** 每次面板数据刷新（手动或自动）成功后，菜单栏文字同步更新。
**影响面：** 这是数据一致性缺陷，优先于所有展示层优化。

### 3.3 警示卡片头部大片空白（#29）

**现状：** 低余量卡片的 accent bar（左侧彩色强调条）作为块级元素占据文档流，在服务名上方产生约 56px 空白。
**目标：** accent bar 沿卡片左边缘全高纵向延伸，不占据额外垂直空间。正常状态卡片不受影响。

## 4. P1 — 应当解决

### 4.1 服务识别方案回退（#4）

**现状：** 卡片头部已能通过纯文字服务名完成识别。
**目标：** 不再额外交付服务品牌图标；卡片头部直接使用服务名文本完成识别，避免重复与授权风险。
**所需资源：** 无新增品牌图标资源需求。

### 4.2 未连接服务的行动引导（#10）

**现状：** 未连接服务显示虚线占位框 + 纯文字描述，无操作入口。
**目标：** 占位卡中加入引导链接（如"前往设置 →"），区分"未安装"与"已安装但未登录"两种状态，给出不同引导文案。

### 4.3 面板底部间距不足（#25）

**现状：** 最后一张卡片紧贴滚动容器底边，无呼吸空间。
**目标：** 滚动容器底部增加适当间距，确保最后一张卡片完整展示且有视觉完成感。

### 4.4 面板与设置 header 设计统一（#28）

**现状：**
- 面板 header：左侧两行（副标题+摘要），右侧两个圆形图标按钮
- 设置 header：左侧单行小字"设置"，右侧"< 返回"文字胶囊按钮

**目标：**
- 返回按钮改为与面板侧一致的圆形图标按钮（仅保留 chevron 箭头，去掉文字）
- 设置 header 左侧可在"设置"标题下增加一行状态反馈（如"已保存"），对齐面板 header 的两行信息密度
- 两个视图的 header 在视觉重心和布局结构上保持对称

### 4.5 设置图标绿色边框移除（#30）

**现状：** 面板 header 中，刷新按钮为灰色中性风格，设置按钮为绿色边框，但绿色无语义含义。
**目标：** 设置按钮统一为与刷新按钮相同的中性灰色风格。

### 4.6 设置选项按使用频率排序（#31）

**现状：** 首屏显示语言、开机自启、刷新间隔——均为一次性/低频设置。
**目标：** 将高频操作（托盘摘要规则、菜单栏服务选择）提前至首屏，低频操作（语言、开机自启）后移。

### 4.7 设置页与面板卡片宽度统一（#32）

**现状：** 设置页卡片因额外水平内边距，比面板卡片窄约 8px。
**目标：** 两个视图中的卡片水平宽度一致，消除切换时的视觉跳动。

### 4.8 设置状态区域展示所有服务（#33）

**现状：** 设置页状态区域仅展示 Codex 会话信息，未展示 Claude Code 连接状态。
**目标：** 遍历所有已配置服务，逐一展示连接状态、数据来源和会话信息。仅使用 Codex 或仅使用 Claude Code 的用户均能看到有效状态。

### 4.9 移除"发送测试通知"按钮（#34）

**现状：** 设置页包含"发送测试通知"按钮，占据空间且为调试功能。
**目标：** 从设置页移除。如需保留调试能力，可通过开发者快捷键或命令行参数触发。

### 4.10 拖拽排序 WKWebView 兼容性（#35）

**现状：** 拖拽使用 HTML5 Drag & Drop API，在 macOS WKWebView 中可能无响应。
**目标：** 改为基于 Pointer Events 的拖拽实现，确保在所有 WebView 引擎中可靠工作。
**验证方式：** 在真实 Tauri macOS 二进制中测试拖拽操作。

### 4.11 菜单栏显示增强（#37，部分回退）

**现状：** 菜单栏仅显示纯文本百分比（如 `37% / 32%`），无法区分服务。
**目标：**
- 根据当前 `menubarService` 设置，在 tooltip 中标明服务名
- 菜单栏图标保持默认应用图标，不再交付 warning / danger 黄红变体

## 5. P2 — 可以解决

### 5.1 无障碍：颜色非唯一状态手段（#11）

**现状：** 进度条仅通过颜色（绿/黄/红）区分状态，色盲用户无法区分。
**目标：** 低余量时在进度条旁或卡片内增加文字标签（如"偏低"、"紧张"）或警告图标。

### 5.2 首次使用引导（#20）

**现状：** 新用户首次打开看到空状态占位卡，不知下一步。
**目标：** 首次启动时显示 2–3 步简洁引导，引导用户连接第一个 AI 服务。
**与 #10 的关系：** #10 解决"单个未连接服务"的行动引导，本项解决"所有服务都未连接"的首次使用引导。两者可共享引导文案。

### 5.3 伪毛玻璃效果决策（#23）

**现状：** CSS 声明了 `backdrop-blur-sm`，但 Tauri 窗口未启用透明/vibrancy，blur 无实际效果。
**目标：** 二选一——
- **方向 A（推荐）：** 去掉伪 blur，改为纯色背景，减少无意义的 GPU 开销
- **方向 B：** 启用 Tauri transparent + macOS vibrancy，实现真正系统级毛玻璃

### 5.4 设置页分组标题间距优化（#26）

**现状：** 去卡片化后，分组标题和下方内容的从属关系不够紧密。
**目标：** 组标题上方间距大于下方间距，拉近标题与其内容的视觉距离，强化从属关系。

### 5.5 代理"应用"按钮降权与标签简化（#27）

**现状：** 代理"应用"按钮使用绿色填充，视觉权重高于所有即时生效设置项。
**目标：**
- 按钮降为中性灰色边框风格（与已降权的通知测试按钮同级），或仅在有未保存更改时才高亮
- 按钮标签从"应用代理"/"Apply proxy"简化为"应用"/"Apply"

### 5.6 英文模式健康摘要截断（#36）

**现状：** 面板 header 摘要文本使用 `truncate`，英文摘要可能因过长被 ellipsis 截断。
**目标：** 缩短英文摘要文案，或调整 header 布局为摘要文本留出更多空间。
