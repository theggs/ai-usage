# 011 — 设置页面重设计方案

**日期：** 2026-03-23
**状态：** 提案

---

## 1. 核心问题诊断

| 问题 | 主面板 | 设置页 |
|------|--------|--------|
| 视觉层级 | 卡片即内容，无多余装饰 | 「分区标题 → 圆角大容器 → 内部控件」三层嵌套，层次过深 |
| 信息密度 | 紧凑、可扫视 | 每个字段独占大量纵向空间，需滚动很远 |
| 风格语言 | 圆角卡片 + 色彩状态 + 进度条 | 传统表单风格，蓝紫色大写字距标题，与主面板毫无呼应 |
| 内容价值 | 每个元素都直接服务用户决策 | 「状态」区展示 data source / session 等调试信息，用户无需关心 |

---

## 2. 布局结构建议

**去掉「分区标题 + 大容器」的双层嵌套**，改为与主面板一致的**扁平卡片列表**——每个设置项就是一张独立的行卡片，竖向排列，卡片间用小间距分隔。不再需要"显示""通用""连接"这些分区概念。

理由：这个应用的设置项总共只有 6-7 项，不需要分组。分组在设置项 > 15 个时才有意义。去掉分组后，用户一屏就能看完所有设置。

**每张设置卡片采用行内布局**：左侧标签，右侧控件，单行完成。类似 iOS Settings 的行样式：

```
┌─────────────────────────────────┐
│  托盘摘要规则          多维摘要 ▾ │
├─────────────────────────────────┤
│  菜单栏服务         Claude Code ▾ │
├─────────────────────────────────┤
│  语言                      中文 ▾ │
├─────────────────────────────────┤
│  开机自启                   ●━━ │
├─────────────────────────────────┤
│  刷新间隔              15 分钟 ▾ │
├─────────────────────────────────┤
│  网络代理            系统代理 ▾  │
└─────────────────────────────────┘
```

这样所有设置一屏可见，不需要滚动，信息密度与主面板匹配。

---

## 3. 具体控件建议

### 3.1 面板顺序

当前的拖拽列表占了很大纵向空间（两个独立拖拽卡片）。建议改为：在同一行内用紧凑的**胶囊标签拖拽排序**，类似标签重排：

```
┌─────────────────────────────────┐
│  面板顺序    [Claude Code] [Codex] │
└─────────────────────────────────┘
```

两个胶囊可左右拖拽交换位置。这把原来 2 个大卡片的纵向空间压缩到 1 行。

### 3.2 刷新间隔

当前用 number input，体验差且容易输入非法值。改为下拉选择器，预设常用选项：`5 分钟 / 10 分钟 / 15 分钟 / 30 分钟`。与其他 select 控件保持一致。

### 3.3 网络代理

当前有一个大的「应用」按钮，视觉上很重。建议：
- 下拉选择「系统代理 / 手动输入 / 不使用代理」
- 仅当选择「手动输入」时，展开一个输入框行
- 去掉「应用」按钮，改为自动保存（与其他设置一致）

### 3.4 开机自启

toggle 开关右对齐在行内即可，不需要独占一块区域。

---

## 4. 卡片视觉风格对齐

设置卡片应复用主面板的视觉语言：
- 同样的圆角（`rounded-2xl`）、边框（`border-slate-200`）、背景（`bg-white`）、阴影
- 标签用 `text-sm font-semibold text-slate-900`，不再用蓝紫色大写字距标题
- 控件（select、toggle）右对齐，保持行的水平节奏感

---

## 5. 「状态」区处理

当前「状态」区展示每个服务的 data source、活跃会话、重置时间等信息。这些是**开发者调试信息**，不是用户需要的设置。

建议：**直接删除状态区**。这些信息已经在主面板的服务卡片上体现了（上次刷新时间、额度状态等）。如果确实需要保留连接诊断能力，可以在最底部加一个不起眼的「诊断信息」文字链接，点击展开折叠面板。

---

## 6. 预期效果

改造后设置页应该：
- **一屏可见**：所有设置项无需滚动（6-7 行卡片 × ~48px ≈ 340px，远小于可用高度）
- **风格统一**：与主面板共享卡片语言，切换时视觉无跳跃
- **操作高效**：每个设置一步到位（选择/切换），无需额外「应用」按钮
- **零认知负担**：不展示用户不理解也不需要的技术信息
