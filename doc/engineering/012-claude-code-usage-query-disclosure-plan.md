# Iteration 012 — Claude Code 用度查询告知与启用控制

**日期：** 2026-03-24
**状态：** 草案
**输入来源：** 用户确认过的 README / 设置页 / 首次启动文案方案、[claude-code-research.md](./claude-code-research.md)、当前设置页与首次启动引导实现

---

## 1. 迭代目标

把 Claude Code 用度查询这类敏感行为从“实现细节”提升为“用户可见、可理解、可控制”的产品能力。

本迭代需要同时解决 3 个问题：

1. 用户需要被明确告知：应用会读取本机现有的 Claude Code 登录凭证，并用它向 Claude 官方接口查询额度状态。
2. 用户需要被明确告知：应用不会存储、修改、管理该凭证，也不会主动将其发送到 AIUsage 自己的服务或其他非官方接口。
3. 用户需要有一个显式开关，能够决定是否启用 Claude Code 用度查询。

---

## 2. 核心决策

### 2.1 用现有首次启动引导承载首屏告知，采用独立卡片确认

本仓库已经有首次启动引导卡片；本迭代不新增 Claude Code 专用 modal，也不把启用开关设计成需要二次确认的阻断式流程。

首次启动告知改为：

- 在现有 onboarding 界面中增加一张独立的 Claude Code 说明卡片
- 卡片带一个确认按钮
- 用户点击后仅关闭这张卡片
- 该确认状态持久化，之后不再重复出现

落实方式：

- README 增加长期透明说明
- 首次启动引导增加 Claude Code 查询说明
- 设置页底部增加 Claude Code 查询卡片与启用开关

### 2.2 设置页原有顺序不变

原有设置项顺序保持不动。Claude Code 相关新增内容统一放在设置页最底部，作为独立卡片收尾。

### 2.3 Claude Code 用度查询默认关闭

新增显式偏好字段后，Claude Code 用度查询默认值为关闭。

原因：

- 当前行为涉及登录凭证读取与外部网络请求
- 在没有用户明确开启前，不应自动触发 Claude Code 查询
- 这能让“说明 + 开关”形成完整闭环

### 2.4 关闭就是硬关闭

当 Claude Code 用度查询关闭时，前后端都不应继续触发 Claude Code 查询链路，包括：

- 不读取 Claude Code 登录凭证
- 不请求 Claude 官方额度接口
- 不产生 Claude Code 面板数据刷新

## 4. 数据模型与行为要求

### 4.1 新增偏好字段

在用户偏好中新增布尔字段：

`claudeCodeUsageEnabled: boolean`

要求：

- 默认值为 `false`
- 进入设置页时，开关状态与该字段一致
- 字段持久化到现有 preferences 存储

为首次启动告知新增持久化字段：

`claudeCodeDisclosureDismissedAt?: string`

要求：

- 默认缺失
- 用户点击 Claude Code 说明卡片确认按钮后写入 ISO 时间戳
- 该字段存在时，onboarding 中不再显示 Claude Code 说明卡片
- 该字段仅控制 Claude Code 说明卡片，不影响现有 onboarding 的显示与关闭逻辑

### 4.2 关闭态行为

当 `claudeCodeUsageEnabled = false` 时：

- 应用初始化阶段不加载 Claude Code 面板状态
- 自动刷新不触发 Claude Code 刷新
- 手动刷新不触发 Claude Code 刷新
- 后端不读取 Claude Code 登录凭证
- 后端不请求 Claude 官方额度接口

### 4.3 开启态行为

当 `claudeCodeUsageEnabled = true` 时：

- Claude Code 查询链路恢复到当前产品能力
- 继续使用现有代理解析逻辑
- 继续沿用现有错误态、限流态、会话恢复态表现

### 4.4 关闭后的偏好归一化

关闭 Claude Code 用度查询后，需要立刻归一化相关偏好，避免留下无效状态。

至少包含：

- 如果当前 `menubarService` 为 `claude-code`，自动回退到 `codex`
- 主面板不再展示 Claude Code 卡片
- 与 Claude Code 相关的选择结果不应继续在 UI 中表现为“仍处于启用中”

说明：

- 本迭代不要求重做现有设置项顺序
- 但必须保证关闭后不存在“已经关闭，却仍被当作当前激活服务”的矛盾状态

## 7. 非目标

本迭代不包含以下内容：

- 不新增 Claude Code 登录或续期能力
- 不新增 Claude Code 权限确认 modal
- 不重排现有设置项顺序
- 不新增新的代理能力，只复用现有代理逻辑
- 不改写 Claude Code 错误态、限流态、会话恢复态文案策略

## 8. 验收标准

1. README 出现独立的 Claude Code 查询说明小节，文案与本说明一致。
2. 首次启动引导中出现独立的 Claude Code 查询说明卡片，且不新增 modal。
3. Claude Code 说明卡片包含确认按钮；点击后该卡片关闭，之后不再重复出现。
4. 设置页最底部出现与主设置卡片同级的 Claude Code 查询卡片。
5. 原有设置项顺序保持不变。
6. Claude Code 开关默认关闭。
7. Claude Code 开关关闭时，应用不会读取 Claude Code 登录凭证，也不会请求 Claude 官方额度接口。
8. Claude Code 开关开启后，现有 Claude Code 用度查询能力恢复正常工作。
9. 关闭开关后，如当前菜单栏服务为 Claude Code，会自动回退到 Codex。
