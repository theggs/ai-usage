## User Scenarios & Testing *(mandatory)*

### User Story 1 - 面板信息准确无冗余 (Priority: P1)

用户点击菜单栏图标打开面板后，进度条内只显示一条百分比文字（无重复），低余量卡片的左侧强调条紧贴卡片边缘（无多余空白），整体信息干净利落，用户可在 1 秒内完成余量扫视。

**Why this priority**: 进度条文字重复（#38）和警示卡片空白（#29）是现有用户直接感知到的视觉缺陷，直接损害产品核心价值——"一眼扫视余量"。属于 P0 级缺陷修复。

**Independent Test**: 打开面板，观察每个进度条是否只有一条百分比文字；将某服务余量设为低于 50%，观察卡片左侧强调条是否沿边缘全高显示且服务名上方无多余空白。

**Acceptance Scenarios**:

1. **Given** 面板已打开且服务有余量数据，**When** 用户查看任意进度条，**Then** 条内仅显示一条百分比文字（如"剩余 47%"），不出现重复数字。
2. **Given** 某服务存在低于 50% 的维度，**When** 面板展示该服务卡片，**Then** 左侧强调条沿卡片左边缘全高纵向延伸，服务名与卡片顶部之间无异常空白。
3. **Given** 进度条填充比例很低（如 10%），**When** 百分比文字位于浅色空白区域上，**Then** 文字仍清晰可读，对比度满足 WCAG AA 标准（4.5:1）。
4. **Given** 所有服务余量均在 50% 以上，**When** 面板展示卡片，**Then** 不出现强调条，卡片布局正常无空白。

---

### User Story 2 - 菜单栏与面板数据始终一致 (Priority: P1)

用户在面板中手动或等待自动刷新后，菜单栏显示的百分比数字与面板中的最新数据一致。菜单栏提供服务名提示，用户无需打开面板即可知道当前菜单栏代表哪个服务。

**Why this priority**: 菜单栏数据不同步（#39）是数据一致性缺陷，用户在面板和菜单栏看到不同数字会产生困惑和不信任。菜单栏展示增强（#37）是产品核心价值"零打断感知余量"的直接延伸。

**Independent Test**: 在面板中点击刷新，验证菜单栏文字立即更新为最新数据；将鼠标悬停在菜单栏图标上，验证 tooltip 包含服务名；warning/danger 状态下菜单栏图标保持默认应用图标。

**Acceptance Scenarios**:

1. **Given** 用户在面板中点击刷新按钮，**When** 刷新成功完成，**Then** 菜单栏显示的百分比数字与面板中最新数据一致。
2. **Given** 后台自动刷新触发，**When** 刷新成功完成，**Then** 菜单栏数据同步更新，无需用户手动操作。
3. **Given** 菜单栏显示百分比数字，**When** 用户将鼠标悬停在菜单栏图标上，**Then** tooltip 中包含当前展示的服务名称。
4. **Given** 某服务最低额度低于 50% 或 20%，**When** 菜单栏显示该服务额度，**Then** 菜单栏标题与 tooltip 仍反映最新数据，但菜单栏图标保持默认应用图标。

---

### User Story 3 - 面板与设置视觉一致 (Priority: P2)

用户在面板和设置页之间切换时，两个视图在 header 布局、按钮风格、卡片宽度等方面保持统一，感受到完整连贯的产品体验。

**Why this priority**: 视觉不一致（#28 header 设计、#30 按钮颜色、#32 卡片宽度）虽不影响功能，但在面板↔设置频繁切换中会累积违和感，降低产品品质感知。

**Independent Test**: 在面板和设置页间反复切换，验证 header 结构对称、按钮风格统一、卡片宽度一致。

**Acceptance Scenarios**:

1. **Given** 用户在面板视图，**When** 查看 header 右侧按钮，**Then** 刷新按钮和设置按钮使用相同的中性灰色圆形图标风格，无颜色差异。
2. **Given** 用户切换到设置页，**When** 查看 header，**Then** 返回按钮为圆形图标按钮（仅 chevron 箭头，无文字），与面板侧按钮风格一致。
3. **Given** 用户在设置页，**When** 查看 header 左侧，**Then** "设置"标题下方有一行状态反馈文字（如"已保存"），与面板 header 的两行信息密度对齐。
4. **Given** 用户在面板和设置页间切换，**When** 观察卡片区域，**Then** 两个视图中的卡片水平宽度一致，无视觉跳动。

---

### User Story 4 - 服务可快速识别与定位 (Priority: P2)

面板卡片直接使用清晰的服务名文本帮助用户快速定位目标服务。未连接的服务提供清晰的行动引导，用户知道下一步该做什么。

**Why this priority**: 清晰、稳定的服务名文本是当前最可靠的识别方式，避免引入不准确或重复的品牌图形。未连接引导（#10）解决用户"看到空状态不知如何行动"的问题。

**Independent Test**: 在面板中验证服务卡片仅显示一次服务名文本且无重复识别徽章；断开某服务后验证占位卡是否有引导操作入口。

**Acceptance Scenarios**:

1. **Given** 面板中有已连接的服务，**When** 用户查看卡片头部，**Then** 卡片使用清晰的服务名文本识别服务，不显示重复的品牌图标或文字徽章。
2. **Given** 某服务未连接，**When** 面板展示该服务的占位卡，**Then** 占位卡中包含引导链接（如"前往设置"），点击可跳转到设置页。
3. **Given** 某服务已安装但未登录，**When** 面板展示占位卡，**Then** 显示与"未安装"不同的引导文案，明确告知用户需要登录。

---

### User Story 5 - 设置页高效易用 (Priority: P2)

设置页首屏展示用户最常调整的选项（摘要规则、菜单栏服务），低频选项后移。状态区域展示所有服务的连接状态。调试功能不再出现在正式界面中。拖拽排序在所有平台上都可靠工作。

**Why this priority**: 设置选项排序（#31）、状态区域补全（#33）、移除测试按钮（#34）、拖拽兼容性（#35）共同影响设置页的实用性和专业感。

**Independent Test**: 打开设置页验证首屏内容排序；检查状态区域是否展示所有服务；确认无"发送测试通知"按钮；在 macOS 上测试拖拽排序。

**Acceptance Scenarios**:

1. **Given** 用户打开设置页，**When** 查看首屏内容（不滚动），**Then** 托盘摘要规则和菜单栏服务选择在可视区域内，语言和开机自启在下方需要滚动才能看到。
2. **Given** 用户同时配置了 Claude Code 和 Codex，**When** 查看设置页状态区域，**Then** 两个服务的连接状态、数据来源均分行展示。
3. **Given** 用户仅使用 Claude Code（未配置 Codex），**When** 查看设置页状态区域，**Then** 显示 Claude Code 的连接状态，不显示"无活动会话"的误导信息。
4. **Given** 用户打开设置页，**When** 浏览全部设置项，**Then** 不存在"发送测试通知"按钮。
5. **Given** 用户在设置页的服务排序区域，**When** 用鼠标按住拖拽手柄并拖动服务到新位置，**Then** 排序操作在 macOS 上可靠完成并立即生效。

---

### User Story 6 - 面板底部和设置分组排版舒适 (Priority: P3)

面板滚动到底部时，最后一张卡片有足够的呼吸空间。设置页分组标题与内容之间的从属关系清晰。代理"应用"按钮视觉权重合理。

**Why this priority**: 底部间距（#25）、分组标题间距（#26）、代理按钮降权（#27）属于排版细节优化，影响产品整体精致感但不影响功能。

**Independent Test**: 在面板中滚动到底部验证间距；在设置页验证分组标题与内容的视觉归属感；检查代理按钮风格。

**Acceptance Scenarios**:

1. **Given** 面板卡片数量超过可视区域，**When** 用户滚动到最底部，**Then** 最后一张卡片下方有明显的底部间距，不紧贴容器边缘。
2. **Given** 用户查看设置页，**When** 观察分组标题（如"通用"、"显示"），**Then** 标题上方间距大于下方间距，标题明确归属于下方内容。
3. **Given** 用户查看代理设置，**When** 代理配置无未保存更改时，**Then** "应用"按钮使用中性灰色边框风格，按钮标签为"应用"/"Apply"。
4. **Given** 用户修改了代理配置但尚未应用，**When** 存在未保存更改，**Then** "应用"按钮可适当提升视觉权重以提示用户。

---

### User Story 7 - 无障碍与引导体验 (Priority: P3)

色觉障碍用户可以通过文字标签（而非仅颜色）判断余量状态。新用户首次打开应用时，有清晰的引导流程帮助连接第一个 AI 服务。面板背景无无效的视觉效果声明。

**Why this priority**: 无障碍（#11）、首次引导（#20）、毛玻璃决策（#23）属于体验完善层，提升产品的包容性和第一印象，但对现有用户日常使用影响较小。

**Independent Test**: 使用色觉模拟工具验证低余量状态是否仅靠颜色区分；以全新状态启动应用验证引导流程；检查面板背景是否有无效的模糊效果声明。

**Acceptance Scenarios**:

1. **Given** 某服务额度低于 50%，**When** 用户查看该服务卡片，**Then** 除颜色外，还有文字标签（如"偏低"）或警告图标辅助表达状态。
2. **Given** 某服务额度低于 20%，**When** 用户查看该服务卡片，**Then** 文字标签明确表达紧急程度（如"紧张"），色觉障碍用户可仅通过文字判断。
3. **Given** 用户首次安装并打开应用，**When** 所有服务均未连接，**Then** 面板显示 2–3 步简洁引导，引导用户前往设置连接第一个 AI 服务。
4. **Given** 面板背景样式，**When** 检查视觉效果，**Then** 不存在无实际效果的伪模糊声明（去掉伪 blur，使用纯色背景）。

---

### User Story 8 - 英文模式健康摘要完整显示 (Priority: P3)

使用英文界面的用户在面板 header 中看到的健康摘要文字完整可读，不被截断。

**Why this priority**: 英文摘要截断（#36）影响国际化用户体验，但中文用户不受影响，优先级最低。

**Independent Test**: 切换为英文界面，在低余量状态下验证 header 摘要文字是否完整显示。

**Acceptance Scenarios**:

1. **Given** 用户使用英文界面，**When** 某服务额度偏低触发健康摘要，**Then** 摘要文字在 header 中完整显示，不被 ellipsis 截断。
2. **Given** 多个服务同时低余量，**When** 摘要需显示最紧急服务信息，**Then** 英文文案足够简短以适应可用空间，同时传达关键信息。

---

### Edge Cases

- 若服务识别元素与服务名文本重复，如何处理？——优先保留服务名文本，不额外增加重复的图标或文字徽章。
- 菜单栏 warning / danger 黄红图标是否继续交付？——不继续交付；菜单栏回退为默认应用图标，仅保留标题与 tooltip 同步。
- 首次引导与未连接引导同时触发时的关系？——首次引导覆盖全面板，完成或跳过后才显示单服务占位卡的行动引导。
- 设置选项排序变更后，已有用户的偏好数据如何迁移？——排序是界面展示层的变更，不影响偏好数据结构，无需迁移。
- 拖拽排序改用 Pointer Events 后，触摸板手势是否仍然支持？——Pointer Events 天然支持鼠标和触摸板，不需额外适配。
- 代理"应用"按钮的"有未保存更改"状态如何判断？——对比当前表单值与已保存的代理配置，有差异则视为未保存。
- 伪毛玻璃去除后，面板与桌面的视觉分离是否足够？——使用纯色背景 + 窗口阴影即可提供足够的层次分离。

## Requirements *(mandatory)*

### Functional Requirements

**面板 - 缺陷修复**

- **FR-001**: 面板进度条内仅显示一条百分比文字，不得出现重复数字。
- **FR-002**: 低余量卡片的左侧强调条沿卡片左边缘全高纵向延伸，不占据额外垂直空间，服务名与卡片顶部之间无异常空白。
- **FR-003**: 进度条内百分比文字在填充区和空白区两种背景上均清晰可读，对比度满足 WCAG AA 标准（4.5:1）。

**菜单栏 - 数据同步与展示增强**

- **FR-004**: 每次面板数据刷新（手动或自动）成功后，菜单栏显示的数字同步更新。
- **FR-005**: 菜单栏图标的 tooltip 包含当前展示的服务名称。
- **FR-006**: 菜单栏在 warning / danger 状态下仍保持默认应用图标，不再要求黄/红状态变体。
- **FR-007**: 本迭代不再要求提供正常、警告、危险三套菜单栏图标资源。

**面板与设置 - 视觉统一**

- **FR-008**: 面板 header 中的设置按钮与刷新按钮使用相同的中性灰色风格，去除无语义的绿色边框。
- **FR-009**: 设置页返回按钮改为圆形图标按钮（仅 chevron 箭头），与面板侧图标按钮风格一致。
- **FR-010**: 设置 header 左侧在"设置"标题下方增加一行保存状态反馈文字（idle/saving/saved/error），成功保存后显示"已保存"并在 2 秒后淡出，失败时持续显示直至下次成功。
- **FR-011**: 面板和设置页中的卡片水平宽度一致。

**面板 - 服务识别**

- **FR-012**: 面板卡片头部使用清晰、可读的服务名文本识别服务。
- **FR-013**: 当服务名文本已足以识别服务时，不额外显示重复的品牌图标、首字母头像或文字徽章。
- **FR-014**: 未连接服务的占位卡包含引导操作入口（跳转到设置页的链接）。
- **FR-015**: 占位卡区分"未安装"和"已安装但未登录"两种状态，分别给出不同引导文案。

**设置页 - 优化**

- **FR-016**: 设置选项按使用频率排序：高频操作（托盘摘要规则、菜单栏服务选择）在首屏，低频操作（语言、开机自启）后移。
- **FR-017**: 设置页状态区域展示所有已配置服务的连接状态、数据来源和会话信息。
- **FR-018**: 从设置页移除"发送测试通知"按钮。
- **FR-019**: 服务拖拽排序改为基于 Pointer Events 的实现，确保在所有 WebView 引擎中可靠工作。
- **FR-020**: 面板滚动容器底部增加不少于 16px 的间距，最后一张卡片不紧贴容器底边。

**设置页 - 排版细节**

- **FR-021**: 设置页分组标题上方间距至少为下方间距的 2 倍（如上方 20px、下方 8px），标题明确归属于下方内容。
- **FR-022**: 代理"应用"按钮默认使用中性灰色边框风格，仅在存在未保存更改时可高亮。按钮标签为"应用"/"Apply"。

**无障碍与引导**

- **FR-023**: 低余量状态除颜色外，辅以文字标签（如"偏低"、"紧张"）或警告图标。
- **FR-024**: 首次启动且所有服务均未连接时，面板显示 2–3 步引导流程，引导用户连接第一个 AI 服务。
- **FR-025**: 面板背景去除无实际效果的伪模糊声明，使用纯色背景。

**国际化**

- **FR-026**: 英文模式下健康摘要文案在 360px 面板 header 可用宽度内完整显示，不得出现 ellipsis 截断。

### Assumptions

- 产品面向 macOS 菜单栏，窗口固定 360x620px，所有布局优化基于此尺寸。
- 当前支持 2 个服务（Claude Code、Codex），设计应考虑未来扩展到 4–5 个服务。
- 当前版本对外发布时不内置第三方品牌图标资源；服务识别以服务名文本为准。
- 菜单栏图标回退为默认应用图标，本迭代不再交付 warning / danger 黄红变体。
- 伪毛玻璃效果采用方向 A（去除），使用纯色背景替代。
- 设置选项排序变更仅影响界面展示层，不改变偏好数据结构。
- "发送测试通知"的调试能力可通过开发者快捷键或命令行参数保留，不在正式界面暴露。
- 拖拽排序仅需支持鼠标和触摸板操作（桌面应用），无需考虑触屏。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 面板中每条进度条内仅有一条百分比文字，100% 无重复。
- **SC-002**: 面板刷新后，菜单栏数据在 1 秒内同步更新，面板与菜单栏显示一致率 100%。
- **SC-003**: 用户打开面板后，可在 1 秒内通过服务名文本识别目标服务，且卡片头部不存在重复识别元素。
- **SC-004**: 低余量卡片无多余空白，强调条与卡片顶部齐平，用户不再反馈"卡片头部有空白"。
- **SC-005**: 在 macOS 真实应用中，拖拽排序操作成功率 100%（当前 HTML5 DnD 在 WKWebView 中存在不可靠问题）。
- **SC-006**: 设置页首屏（不滚动）可见托盘摘要规则和菜单栏服务选择两个高频设置。
- **SC-007**: 色觉模拟测试（红绿色盲模式）下，用户仅通过文字标签即可判断余量状态，不依赖颜色。
