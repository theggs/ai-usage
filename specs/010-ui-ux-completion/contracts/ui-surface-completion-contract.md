# UI 契约：菜单栏、面板、设置页与首次引导补完

**特性**：010-ui-ux-completion  
**日期**：2026-03-23

> 变更说明（回退）：
> 菜单栏 warning / danger 黄红状态图标需求已放弃。菜单栏契约仅保留标题与 tooltip 同步，图标统一使用默认应用图标。

本文档定义了 010 completion 特性中受影响的菜单栏、面板、设置页与首次引导表面的稳定用户可见行为。

---

## 1. 菜单栏表面契约

### 输入

- `preferences.menubarService`
- `preferences.traySummaryMode`
- 最近一次刷新成功后生成的规范 `PanelPlaceholderItem[]`

### 行为

- 菜单栏标题在每次手动刷新成功后都必须更新。
- 菜单栏标题在每次自动刷新成功后都必须更新。
- 菜单栏 tooltip 必须包含当前所代表的服务名称。
- 菜单栏图标必须保持默认应用图标，不再根据 warning / danger 状态切换黄红变体。

### 图标行为映射

| 最低剩余百分比 | 菜单栏图标行为 |
|----------------|----------------|
| `> 50` | 默认应用图标 |
| `20 - 50` | 默认应用图标 |
| `< 20` | 默认应用图标 |
| 无已连接 item | 默认应用图标 |

### Tooltip 示例

| 选中服务 | 摘要 | Tooltip |
|----------|------|---------|
| `codex` | `37%` | `AIUsage · Codex · 37%` |
| `claude-code` | `18%` | `AIUsage · Claude Code · 18%` |
| empty | 无 | `AIUsage · Codex` |

## 2. 面板卡片契约

### 额度行

- 每条额度行必须且只能渲染一条百分比标签。
- 百分比标签在填充区和未填充区上都必须保持清晰可读。
- warning 与 danger 额度行必须提供非颜色的严重程度提示，例如文本（`偏低` / `紧张`）或告警图标。

### 卡片紧急程度表现

- warning 与 danger 卡片必须在左侧展示一条沿整张卡片高度延展的强调条。
- 强调条不得参与文档流，不得在服务标题上方制造空白。
- healthy 卡片必须使用中性卡片样式，且不渲染强调条。

### 服务识别

- 已连接卡片必须使用清晰的服务名文本识别服务。
- 当服务名文本已能完成识别时，界面不得再追加重复的服务图标、首字母头像或文字徽章。

---

## 3. 空状态与首次引导契约

### 首次引导

- 当所有服务均未连接，且首次引导尚未被关闭时，面板必须显示一个 2-3 步的首次引导覆盖层。
- 首次引导覆盖层必须先于按服务占位卡显示。
- 首次引导覆盖层必须提供跳转设置页的操作入口。
- 当用户跳过首次引导时，关闭状态必须持久化到现有偏好存储中。

### 按服务占位引导

- 首次引导关闭或完成后，未连接服务必须显示自己的占位卡。
- 占位卡至少要区分：
  - `not-installed`
  - `signed-out`
  - 通用 `disconnected`
- 每张占位卡都必须包含前往设置页的引导入口。

---

## 4. 设置页布局契约

- 设置页首屏必须优先展示高频控制项：
  - 托盘摘要模式
  - 菜单栏服务
- 语言、开机自启等低频控制项必须下移到首屏之后。
- 设置页返回操作必须与面板 header 操作按钮共用相同的中性圆形图标按钮样式。
- 设置页标题区域必须包含一行副状态文案，使其信息密度与面板 header 对齐。
- 即时保存生效时，header 副文案必须短暂显示"保存中…"，成功后切换为"已保存"并在 2 秒后淡出。
- 保存失败时副文案必须显示"保存失败"，持续到下次成功保存。
- 代理"应用"操作遵循相同的状态反馈规则。
- 设置页中的卡片/分组横向宽度必须与面板卡片对齐。
- 正式设置页中不得再渲染通知测试按钮。

---

## 5. 设置页状态契约

- 状态区域必须按受支持服务渲染归一化状态卡片，每个服务一张。
- 每张状态卡必须至少暴露：
  - 服务名称
  - 连接真实状态（`connected`、`disconnected`、`stale`、`failed` 或 empty/setup）
  - 数据来源
  - 会话或快照细节（如存在）
- 单服务场景下不得出现与缺失服务相关的误导性 fallback 文案。

---

## 6. 服务排序契约

### 交互

- 排序必须采用 Pointer 驱动交互，而不是 HTML5 drag/drop。
- 每一行都必须展示可见的拖拽手柄。
- 拖拽过程中必须预览新顺序，而不是只在 drop 后变化。

### 持久化

- 松手完成 drop 后必须立即持久化 `serviceOrder`。
- 持久化成功后，面板必须无需额外操作即可反映新顺序。
- 持久化失败时，界面必须回退到最后一次确认成功的顺序，并展示真实错误反馈。

---

## 7. 国际化契约

- 英文模式下的面板 header 健康摘要必须在固定宽度内完整显示，不允许通过省略号解决。
- 中英文紧急状态表达必须语义对齐，即使具体文案长度不同。

---

## 8. 视觉表面契约

- 面板背景必须使用纯色表面风格，而不是伪 blur 声明。
- 面板滚动容器在最后一张卡片后方必须保留不少于 16px 的底部间距。
- 设置页分组标题上方间距必须至少为下方间距的 2 倍，确保标题在视觉上归属于下方内容。
- 代理“应用”按钮默认使用中性样式，只有在代理草稿存在未保存变更时才提升强调级别。
