# UI 契约：优惠活动提示

**特性**：013-promotion-status  
**日期**：2026-03-25

本文档定义“主界面头部优惠活动提示”这一桌面表面的稳定用户可见行为。该特性不新增宿主命令契约，重点约束头部提示区、默认胶囊视图、同区域锚定的完整状态浮层与多服务状态语义。

---

## 1. 输入

- 现有头部主信息（由当前面板主状态驱动）
- `VisibleServiceScope`
- `PromotionDisplayDecision`
- 当前语言文案

---

## 2. 头部结构契约

- 头部左侧必须采用两级信息结构：
  - 第一行：现有主信息，不属于本特性的新增范围
  - 第二行：优惠活动提示，使用更小字号
- 第二行必须位于标题区下方，不得扩展成独立卡片或说明段落
- 第二行默认应以多个并列小胶囊承载服务状态，每个胶囊包含服务图标与极短状态文案
- 第二行不得额外显示固定可见前缀“促销信息：”
- 右侧刷新/设置按钮的既有布局不得因本特性变成新的多行按钮区

---

## 3. 默认聚焦视图契约

### 进入条件

- 面板打开时默认进入 `focused` 模式

### 展示规则

- 默认只显示以下服务状态：
  - `正在优惠时段`
  - `优惠资格待确认`
- 以下状态默认不显示：
  - `不在优惠时段`
  - `无优惠活动`
- 若默认视图隐藏了部分服务，第二行本身必须可触发完整状态浮层
- 状态语义不能只靠颜色表达；默认胶囊必须同时包含服务图标与短文案，颜色只作辅助

### 否定态

- 当没有任何服务处于“有优惠/待确认”状态时，第二行必须显示统一否定态文案，如“当前无优惠活动”

---

## 4. 查看全部视图契约

### 进入方式

- “查看全部”必须在主界面内完成
- 默认由促销信息行本身在 hover 时触发预览
- hover 不能是唯一进入方式；必须同时支持 focus 或 click 的等效触发
- hover / focus 只进入临时预览态
- click 进入稳定展开态
- 不再额外放置显式 `...` 入口或其他高权重主按钮

### 展示规则

- 进入 `all` 模式后，应显示所有当前可见服务的促销状态
- `all` 模式必须以锚定在标题下方同一提示区域的浮层完成，不得渲染到新的独立区域、附加块或说明段落
- 稳定展开态必须可通过点击浮层外区域或按 `Esc` 关闭
- 浮层中的每个服务项必须使用“两行信息块”展示，而不是复用默认胶囊的极简表达
- 第一行必须展示“图标 + 服务名 + 更完整结论 + 可选优惠幅度”
- 第二行必须展示时间信息或时段说明
- Claude Code 的第二行应展示“活动日期范围 + 用户本地优惠窗口”，例如 `2026.03.13-2026.03.28 · 本地优惠窗口`
- Codex 的第二行应展示“当前持续优惠时段”，不使用 `00:00-23:59` 这类机械时间串
- 在 `all` 模式下，以下状态必须可见：
  - `正在优惠时段`
  - `不在优惠时段`
  - `优惠资格待确认`
  - `无优惠活动`

---

## 5. 服务状态语义契约

| 状态 | 含义 | 可否默认显示 |
|------|------|--------------|
| `正在优惠时段` | 当前明确命中公开定义或产品认可的优惠时段 | 是 |
| `不在优惠时段` | 存在时段型活动，但当前不在优惠窗口内 | 否 |
| `优惠资格待确认` | 活动可能相关，但用户资格未知 | 是 |
| `无优惠活动` | 当前无适用活动 | 否 |

约束：
- 不得把“优惠资格待确认”伪装成“正在优惠时段”或“无优惠活动”

---

## 6. 多服务与多活动契约

- 当多个服务同时处于可显示状态时，第二行必须以聚合方式表达多个服务，而不是只给出单一服务结论
- 当同一服务下存在多个活动时，最终只展示一个服务级结论
- 同一服务的结论优先级：
  - `正在优惠时段`
  - `不在优惠时段`
  - `无优惠活动`
- 若资格未知，则该服务优先展示为 `优惠资格待确认`

---

## 7. 国际化与可读性契约

- 中英文两套文案都必须支持上述四类状态
- 默认聚焦视图必须保持一行紧凑表达
- 当前版本按现有已支持服务数量优化；若内容在真实壳层宽度下无法稳定承载，应优先压缩服务简称和状态短语，而不是升级为说明段落或拆到新的信息区
- 两行信息块中的第二行可以承载更多信息，但仍必须保持低打扰和可扫读，不得退化为长段落说明
- 完整状态浮层在两种语言下都必须保持低打扰、可发现且不易抖动

---

## 8. 真实状态契约

- 对当前 Codex 活动，产品语义按“24 小时都处于优惠时段”处理，可在浮层中展示明确优惠幅度 `2x`
- 用户资格未知时，必须显示“优惠资格待确认”
- 活动结束后不得继续在头部中作为当前活动出现
- 历史活动只服务于目录留痕，不直接进入当前头部展示
