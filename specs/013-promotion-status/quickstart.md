# 快速开始：优惠活动提示

**特性**：013-promotion-status  
**日期**：2026-03-25

## 前置条件

- 通过 `nvm use` 使用 Node.js 20 LTS
- 已安装 Rust stable toolchain
- 具备 macOS 环境，以便验证真实菜单栏面板宽度与头部一行聚合提示
- 如需验证真实日期边界，建议准备一组固定时间样例或测试桩

## 开发工作流

```bash
# 前端 + 宿主壳层
npm run tauri:dev

# 前端单元/集成测试
npm test

# 真实壳层与截图检查
npm run test:e2e:tauri
npm run test:e2e:screenshots
```

> 说明：本特性默认以前端本地促销目录与 UI 解析为主。若实现过程中没有修改 Rust 宿主层，则 `cargo test` 不是首要验收项；若后续实际改动跨到宿主或共享契约，再补跑 `cargo test`。

## 验证清单

### 1. 默认聚焦视图

1. 打开主面板，确认头部左侧保留两级信息结构。
2. 确认第二行只显示当前“正在优惠时段/优惠资格待确认”的服务。
3. 确认无优惠服务不会在默认视图中挤占空间。

### 2. 完整状态浮层

1. 在默认视图下确认第二行以小胶囊展示当前“正在优惠时段/优惠资格待确认”的服务。
2. 将鼠标悬停到促销信息行，确认完整状态浮层出现。
3. 使用键盘 focus 再验证一次，确认完整信息不是 hover-only。
4. 使用 click 打开稳定展开态，再验证点击外部区域与按 `Esc` 都能关闭浮层。
5. 确认完整状态仍锚定在标题下方同一提示区域，而不是跳到新的独立信息区。
6. 确认每个服务在完整浮层中都以“两行信息块”展示：第一行是服务名/结论/幅度，第二行是时间信息。
7. 确认 Claude Code 的第二行显示“活动日期范围 + 用户本地优惠窗口”，Codex 的第二行为“当前持续优惠时段”。
8. 确认无优惠服务也能被看见，且状态表达真实。

### 3. 时间与时区边界

1. 使用固定样例验证 Claude 工作日优惠窗口内外两种情况。
2. 验证跨时区换算后，窗口判断与预期一致。
3. 验证 Codex 按当前产品定义显示为“正在优惠时段”，并在完整浮层中带出 `2x` 幅度。

### 4. 资格未知与否定态

1. 构造用户资格未知样例。
2. 确认该服务显示“优惠资格待确认”。
3. 构造所有服务都无当前优惠的样例。
4. 确认头部回退为统一否定态文案。

### 5. 国际化与一行可读性

1. 切换为中文，确认默认聚焦视图为一行紧凑表达。
2. 切换为英文，确认默认聚焦视图仍保持可读且不退化为长段落。
3. 确认胶囊短文案在两种语言下都不喧宾夺主，且无需依赖固定“促销信息：”前缀。

### 6. 真实壳层视觉检查

1. 在真实 Tauri 面板中打开主界面。
2. 确认促销副文案与现有右侧按钮无重叠。
3. 确认 hover 预览、click 稳定展开、外部点击/Esc 关闭前后，头部没有明显视觉跳动、抖动或溢出。

## 回归重点

- 促销规则不得散落在 `AppShell` 中硬编码
- Codex 当前促销应按“24 小时优惠时段”处理，并在完整浮层中显示 `2x`
- 未知资格必须显示为中性状态，而不是被隐藏或误判
- 默认聚焦视图必须保持一行紧凑表达
- 完整信息不能只靠 hover 发现，focus/click 必须等效可用
- 胶囊颜色不能成为唯一状态语义
