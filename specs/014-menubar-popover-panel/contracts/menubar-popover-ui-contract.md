# Menubar Popover UI Contract：吸附式弹出面板

## Purpose

定义 AIUsage 主面板从“普通窗口”升级为“菜单栏吸附式弹出面板”后的稳定用户可见契约，包括 tray 触发、窗口定位、关闭路径、视觉结果与异常回退。

## Host Commands

本特性默认不新增新的 `invoke` 命令族，继续复用现有面板/偏好命令：

| Command | Input | Output | Behavior |
|--------|--------|--------|--------|
| `get_codex_panel_state` | None | 主面板状态 | 返回 panel 初始渲染所需的归一化数据 |
| `refresh_codex_panel_state` | None | 更新后的主面板状态 | 刷新 Codex 快照并返回真实状态 |
| `get_claude_code_panel_state` | None | 主面板状态 | 返回 Claude Code 面板数据 |
| `refresh_claude_code_panel_state` | None | 更新后的主面板状态 | 刷新 Claude Code 快照并返回真实状态 |
| `get_preferences` | None | 用户偏好 | 返回语言、刷新间隔、服务顺序等偏好 |
| `save_preferences` | 偏好 patch | 更新后的用户偏好 | 持久化设置改动并返回规范化结果 |
| `set_autostart` | `enabled` boolean | 更新后的用户偏好 | 应用开机自启设置并返回真实持久化结果 |
| `get_runtime_flags` | None | 运行时标记 | E2E 模式下可用于切换测试行为 |

## Window Behavior Contract

- tray 左键点击负责切换 `main` 窗口的显示/隐藏状态。
- 当主面板从隐藏变为显示时，宿主必须先完成位置决策，再显示窗口。
- 正常场景下，主面板必须以菜单栏图标水平中心线为基准，在其下方弹出。
- 当中心对齐结果即将越界时，宿主必须把主面板收敛到当前显示器可视工作区内。
- 当本次无法获取菜单栏图标精确位置时，宿主必须优先回退到上一次成功弹出的位置。
- 当既没有本次 tray 几何信息，也没有有效历史位置时，宿主必须采用当前显示器顶部安全默认位，确保窗口仍可见。
- 原生关闭请求必须被拦截并转换为隐藏主窗口，而不是退出进程。
- 主窗口失去焦点时必须自动隐藏。
- 当主面板处于显示状态且用户按下 `Esc` 时，主面板必须隐藏。
- 主窗口保持不可调整大小，并以菜单栏面板 footprint 运行。

## UI Expectations

- 主面板视觉上只能识别到一个白色面板本体；不得再出现当前这种白色卡片外侧的灰色、渐变或其他明显衬底背景。
- 主面板不得显示系统标题栏、交通灯按钮或独立窗口标题。
- 重新打开主面板时，UI 必须默认回到 `panel` 视图，而不是停留在 `settings` 或其他次级状态。
- panel 与 settings 仍共享同一个窗口与壳层，不新增第二个设置窗口。
- 内容较少时，主面板高度应随内容收紧，不保留明显多余空白。
- 内容较多时，主面板高度应受当前显示器工作区上限约束，并通过面板内部滚动访问超出内容。
- 本特性不新增用户可见的手动拖动窗口交互。

## Positioning Contract

| Scenario | Expected Behavior |
|--------|--------|
| 当前显示器空间充足 | 以 tray 图标水平中心线为基准，在其下方弹出 |
| 图标靠近左/右边缘 | 先按中心线对齐，再向工作区内收敛，不能横向越界 |
| 多显示器点击 | 主面板出现在本次点击所在显示器上 |
| 本次 tray 图标几何不可用 | 回退到上一次成功弹出的位置 |
| tray 几何与历史位置都不可用 | 使用当前显示器顶部安全默认位 |

## Error / Fallback Contract

| Scenario | Expected UI / Host Response |
|--------|--------|
| 本次 tray 图标位置不可读取 | 不随机跳位，优先使用上一次成功位置 |
| 重新打开前用户停留在设置页 | 主面板显示时重置回 panel 视图 |
| 内容高度超过工作区上限 | 保持窗口在工作区内，并启用内部滚动 |
| 用户通过 `Esc` 关闭主面板 | 行为等同于点击外部关闭；再次点击 tray 图标可正常打开 |
| 主面板隐藏后重新打开 | 继续保留已有数据状态，但 UI 壳层回到主面板默认视图 |

## Out of Scope

- 新增第二个原生窗口或拆分设置为独立窗口
- 引入真正透明窗口所需的 macOS 私有 API 作为默认方案
- 新增用户可见的窗口拖动、停靠记忆或自由摆位能力
- 改造底层服务数据来源、快照结构或偏好持久化格式
