# Quickstart：菜单栏吸附式弹出面板

## 目标

验证 AIUsage 主窗口已从传统窗口样式升级为菜单栏吸附式弹出面板：点击菜单栏图标后从对应图标下方弹出、视觉上只保留单一白色面板本体、重新打开回到主额度面板，并在真实桌面环境中保持稳定。

## 前置条件

- Node.js 24 LTS 已安装
- Rust stable toolchain 已安装
- Tauri 桌面依赖已可正常运行
- macOS 桌面会话处于真实可交互状态：显示器已唤醒、已解锁、可执行前台 GUI 测试
- 在运行任何 GUI/E2E/截图验证前，先确认当前会话不是纯 CLI 远程环境

## GUI 会话预检查

1. 运行 `pmset -g assertions`，确认当前会话不是明显的“用户不活跃/显示器睡眠”状态。
2. 如需长时间截图或 E2E，可使用 `caffeinate -d` 保持显示器唤醒。
3. 若终端或运行环境缺少辅助功能权限，先为相关终端应用授予 macOS Accessibility 权限。

## 命令

```bash
npm test
cargo test
npm run test:e2e:tauri
npm run test:e2e:screenshots
```

## 推荐验证顺序

1. 先运行 `npm test` 和 `cargo test`，确保前端壳层与宿主定位函数回归通过。
2. 再运行 `npm run test:e2e:tauri`，验证真实 Tauri 主窗口行为。
3. 最后运行 `npm run test:e2e:screenshots`，确认视觉结果与边缘场景截图。

## 手工验证清单

### 菜单栏定位与收放

1. 点击菜单栏图标，确认主面板从图标下方弹出，而不是从任意旧窗口位置出现。
2. 再次点击同一菜单栏图标，确认主面板隐藏。
3. 打开主面板后点击外部区域，确认主面板隐藏。
4. 打开主面板后按下 `Esc`，确认主面板隐藏。
5. 关闭后再次点击菜单栏图标，确认主面板正常重新打开。
6. 如主面板在设置页或促销预览浮层状态下按下 `Esc`，确认收起的是整个主面板；再次打开后回到主额度面板。

### 视觉结果

1. 确认不再出现系统标题栏、交通灯按钮或独立窗口标题。
2. 确认视觉上只看到一个白色面板本体，不再看到白色卡片外侧的灰色/渐变衬底区域。
3. 确认主面板整体更接近菜单栏弹出面板，而不是独立桌面窗口。

### 尺寸与滚动

1. 在 panel 内容较少场景下，确认主面板高度会收紧，不存在明显多余空白。
2. 切换到 settings 或制造较长内容场景，确认主面板不会超出当前显示器可视工作区。
3. 在常见双服务主面板场景下，确认主内容尽量可首屏看全，不会因为默认高度过短而必须先滚动才能看完整体状态。
4. 在内容超出上限时，确认通过主面板内部滚动访问剩余内容，而不是让窗口本身越界。

### 多显示器与边缘

1. 在主显示器菜单栏点击图标，确认主面板出现在主显示器。
2. 在第二显示器菜单栏点击图标，确认主面板出现在第二显示器。
3. 将图标置于靠近左侧或右侧边缘的场景，确认主面板仍完整保留在可视工作区内。

### 视图生命周期

1. 打开主面板，进入设置页，然后关闭主面板。
2. 再次打开主面板，确认默认回到主额度面板，而不是停留在设置页。
3. 确认返回主面板后，刷新、设置入口和现有服务卡片仍可正常操作。

## 回归关注点

- tray 文本与图标状态仍按现有逻辑更新
- 失焦隐藏仍然生效，不会因为新定位逻辑而回退
- 主面板默认高度足以覆盖常见主面板内容，不出现“过去可一眼看全、现在必须滚动”的体验回退
- `currentView`、overlay 和滚动状态在重新打开后不会残留错误中间态
- 不新增新的偏好文件或位置持久化文件
- 现有 `tests/e2e/tauri-driver.mjs` 与 Swift 辅助脚本仍能定位到真实窗口
