# Feature Specification: 菜单栏 UI/UX 全面改进

**Feature Branch**: `004-menubar-ui-overhaul`
**Created**: 2026-03-19
**Status**: Draft
**Input**: AIUsage 菜单栏程序 UI/UX 全面改进：修复窗口关闭退出进程、精简主面板信息层级、添加额度进度条与颜色分级、补全设置页国际化、视觉精简、交互打磨

## Clarifications

### Session 2026-03-19

- Q: 窗口失焦行为——切换应用时窗口是否自动隐藏？ → A: Popover 模式，任何失焦都自动隐藏（点击外部、切换应用均触发）
- Q: 简化主面板后的错误/离线状态如何展示？ → A: 状态行内联，在"上次刷新时间"行中附加错误提示文字和颜色变化

### Session 2026-03-20 (实机验收反馈)

- 进度条在实际界面中视觉存在感不足，需要确保在各种百分比下清晰可见
- 后端（Rust 宿主层）返回的数据字符串（badgeLabel、remainingAbsolute、resetHint、statusMessage、sessionLabel）未做本地化，中文模式下仍显示英文 → 已通过前端本地化工具函数解决（`localizeRemaining`、`localizeResetHint`、`localizeBadgeLabel`），后端保持英文输出，前端负责映射
- 主面板状态行中"托盘摘要预览"信息对普通用户无意义，应移到设置页
- 开机自启 toggle switch 视觉尺寸偏小，需增强可见性
- 额度维度中"剩余 X%"文本与进度条标题行的百分比数字重复，移除冗余文本行，每个维度简化为三行：标签+百分比、进度条、重置提示
- Tailwind CSS 从 v3 (PostCSS) 迁移至 v4 (Vite 插件 + `@source` CSS 指令)，需 Node.js >= 20

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 快速查看额度剩余 (Priority: P1)

用户点击菜单栏图标，弹出窗口立即展示各服务的额度剩余情况。核心信息（剩余百分比、重置时间）一目了然，无需在大量元信息中寻找。每个额度维度以三行紧凑布局呈现：标签+百分比、进度条、重置提示。进度条和颜色分级让用户瞬间判断额度健康状态。

**Why this priority**: 这是菜单栏应用的核心使用场景——用户打开它只为快速看一眼额度还剩多少。当前信息被三块重复的状态区域淹没，核心价值未被有效传达。

**Independent Test**: 点击菜单栏图标，在 2 秒内能判断出哪个额度维度最紧张、剩余多少。

**Acceptance Scenarios**:

1. **Given** 用户点击菜单栏图标，**When** 弹出窗口显示，**Then** 主面板仅展示：标题行、额度卡片列表（含进度条）、上次刷新时间（一行小字）、刷新按钮，无多余的状态信息区块；状态行不包含"托盘摘要预览"等调试/高级信息
2. **Given** 某额度维度剩余 2%，**When** 用户查看该维度，**Then** 进度条显示为红色/警告色，与剩余 100% 的绿色有明显视觉区分；进度条高度和颜色对比度足够在一眼扫视中被感知
3. **Given** 某额度维度剩余 35%，**When** 用户查看该维度，**Then** 进度条显示为黄色/中间态颜色
4. **Given** 某额度维度剩余 80%，**When** 用户查看该维度，**Then** 进度条显示为绿色/健康色

---

### User Story 2 - 关闭弹出窗口后程序继续运行 (Priority: P1)

用户关闭弹出窗口（点击红色关闭按钮或失焦），程序继续在菜单栏运行，不退出进程。下次点击菜单栏图标时窗口重新出现。

**Why this priority**: 这是菜单栏应用的基本行为规范。当前行为（关闭 = 退出）是功能性 bug，破坏了用户对 macOS 菜单栏应用的基本预期。

**Independent Test**: 关闭窗口后，菜单栏图标仍在，再次点击可重新打开。

**Acceptance Scenarios**:

1. **Given** 弹出窗口处于打开状态，**When** 用户点击窗口关闭按钮，**Then** 窗口隐藏但菜单栏图标仍存在，进程不退出
2. **Given** 窗口已被关闭隐藏，**When** 用户再次点击菜单栏图标，**Then** 窗口重新显示，数据状态保持不变
3. **Given** 弹出窗口处于打开状态，**When** 用户点击窗口外的其他区域或切换到其他应用（如 Cmd+Tab），**Then** 窗口自动隐藏（Popover 模式，任何失焦均触发）

---

### User Story 3 - 使用中文界面无英文残留 (Priority: P2)

用户将语言设为中文后，应用中所有用户可见文本均为中文，包括前端 UI 标签、后端返回的数据字符串（如额度维度的剩余量描述、重置提示、徽章标签、会话标签、状态消息等）。

**Why this priority**: 国际化不完整会让应用显得粗糙、不专业，影响用户信任感。当前后端返回的数据字符串（如 "100% remaining"、"Resets in 5h"、"Live"、"Local Codex CLI"）在中文模式下仍为英文，占界面可见文本的相当比例。

**Independent Test**: 语言设为中文后，逐项检查所有界面文本（含额度卡片内的数据文本和设置页的会话信息），不应出现任何英文。

**Acceptance Scenarios**:

1. **Given** 语言设为中文，**When** 用户进入设置页，**Then** 所有标签（"语言"、"刷新间隔"、"开机自启"、"偏好设置"、"操作"等）均为中文
2. **Given** 语言设为中文，**When** 用户查看额度卡片，**Then** "Last refreshed:" 等前端标签替换为对应中文
3. **Given** 语言设为中文，**When** 用户查看额度卡片内的数据文本，**Then** 徽章标签（如 "Live" → "实时"）、剩余量描述（如 "100% remaining" → "剩余 100%"）、重置提示（如 "Resets in 5h" → "5 小时后重置"）均为中文
4. **Given** 语言设为中文，**When** 用户查看设置页 Codex CLI 区块，**Then** 会话标签（如 "Local Codex CLI" → "本地 Codex CLI"）和状态消息（如 "Live Codex limits available..." → 对应中文）均为中文
5. **Given** 语言设为英文，**When** 用户查看所有界面，**Then** 所有文本均为英文，无中文残留

---

### User Story 4 - 视觉风格贴合 macOS 菜单栏应用 (Priority: P2)

弹出窗口的视觉风格紧凑、轻量，与 macOS 原生菜单栏应用（如系统偏好设置弹窗、iStat Menus）的信息密度和视觉节奏一致。

**Why this priority**: 当前过重的圆角、多层玻璃拟态和大渐变背景让应用看起来像独立窗口应用而非菜单栏工具，影响使用效率。

**Independent Test**: 弹出窗口打开后，视觉上与其他 macOS 菜单栏应用的密度和风格协调。

**Acceptance Scenarios**:

1. **Given** 弹出窗口显示，**When** 用户观察界面，**Then** 圆角大小统一且适中（不超过 16px），无过大的装饰性圆角
2. **Given** 弹出窗口显示，**When** 用户观察卡片层级，**Then** 嵌套层级不超过两层（容器 > 卡片），无三层嵌套的视觉深度
3. **Given** 弹出窗口显示，**When** 用户观察窗口尺寸，**Then** 窗口宽度在 320-380px 范围，高度根据内容自适应或不超过 620px

---

### User Story 5 - 交互反馈清晰及时 (Priority: P3)

用户执行操作（刷新、保存设置、测试通知）时，界面给出即时、明确的反馈，不存在操作后无反应或反馈不可见的情况。

**Why this priority**: 交互反馈是基本的可用性要求，但不影响核心功能。

**Independent Test**: 逐项执行刷新、保存、测试通知操作，每次操作后应有可见的状态反馈。

**Acceptance Scenarios**:

1. **Given** 用户点击刷新按钮，**When** 数据正在加载，**Then** 按钮显示加载状态（如 spinner）并禁用，防止重复点击
2. **Given** 用户在设置页保存偏好，**When** 保存成功，**Then** 保存按钮临时变为成功状态（如文字变为"已保存"），无需滚动即可看到反馈
3. **Given** 设置页中 Autostart 开关，**When** 用户查看，**Then** 控件为 toggle switch 样式，尺寸足够醒目（宽度不小于 44px、高度不小于 24px），开/关状态颜色对比明显，与页面其他控件视觉风格一致
4. **Given** 设置页中"保存"和"测试通知"按钮，**When** 用户查看，**Then** 它们分属不同的功能区块，不并列在同一 "Actions" 组中

---

### Edge Cases

- 用户在数据加载完成前关闭窗口，再次打开时应继续加载或显示上次缓存的数据
- 额度百分比为 0% 时，进度条应完全为空且使用最强烈的警告色
- 额度百分比数据缺失（undefined）时，进度条应显示为灰色占位状态
- 窗口高度超出屏幕可用空间时，应支持滚动
- 同步错误发生时，状态行文字变为错误提示色；错误恢复后，状态行自动恢复正常颜色

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在用户关闭弹出窗口时隐藏窗口而非退出进程
- **FR-002**: 系统 MUST 在用户点击菜单栏图标时切换窗口显示/隐藏状态
- **FR-002a**: 系统 MUST 在窗口失去焦点时（点击外部区域、切换应用等）自动隐藏窗口（Popover 模式）
- **FR-003**: 主面板 MUST 仅展示：标题行、额度卡片列表、状态行（上次刷新时间 + 内联错误提示）、刷新按钮，移除冗余的状态信息区块（深绿色状态栏、蓝色同步信息卡）
- **FR-003a**: 当同步出错或数据获取失败时，状态行 MUST 在"上次刷新时间"旁内联显示错误提示文字，并以颜色变化（如警告色）引起用户注意
- **FR-003b**: 主面板状态行 MUST NOT 包含"托盘摘要预览"等调试/高级信息；托盘摘要预览信息 SHOULD 移至设置页展示
- **FR-004**: 每个额度维度 MUST 包含进度条，按比例填充以直观表示剩余百分比；进度条高度 MUST 不小于 8px，颜色与背景对比度足够一眼可辨
- **FR-005**: 进度条颜色 MUST 根据剩余百分比分级：>50% 为健康色、20%-50% 为警告色、<20% 为危险色
- **FR-006**: 所有用户可见文本 MUST 根据语言设置动态切换，不存在硬编码的语言文本；此要求覆盖前端 UI 标签和后端（Rust 宿主层）返回的数据字符串
- **FR-006a**: 后端返回的额度数据字符串（badgeLabel、remainingAbsolute、resetHint）MUST 在前端根据用户语言偏好进行本地化转换；前端通过 `localizeRemaining`、`localizeResetHint`、`localizeBadgeLabel` 等工具函数解析后端英文字符串并映射为当前语言的文本
- **FR-006b**: 后端返回的会话信息字符串（statusMessage、sessionLabel）MUST 在前端通过 i18n 字典进行本地化映射
- **FR-007**: 界面圆角 MUST 统一在 12-16px 范围，移除 28px/32px 的装饰性圆角
- **FR-008**: 卡片嵌套层级 MUST 不超过两层
- **FR-009**: 刷新按钮 MUST 在加载状态时显示加载指示并禁用，防止重复触发
- **FR-010**: 保存设置后的成功反馈 MUST 在保存按钮附近可见，无需滚动
- **FR-011**: Autostart 控件 MUST 使用 toggle switch 样式替代原生 checkbox
- **FR-012**: 设置页中"保存偏好"和"测试通知" MUST 分属不同的功能分区
- **FR-013**: 弹出窗口 MUST 不可由用户调整大小
- **FR-014**: 原主面板中的"活跃会话"、"数据来源"等元信息 MUST 保留在设置页的 Codex CLI 区块中

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户从点击菜单栏图标到判断出最紧张的额度维度，所需时间不超过 2 秒
- **SC-002**: 关闭弹出窗口后再次打开，100% 的情况下程序未退出、数据状态保持
- **SC-003**: 语言设为中文时，界面中硬编码英文文本数量为 0
- **SC-004**: 主面板信息区块数量从当前的 3 块（状态栏 + 同步卡 + 卡片列表）减少到 1 块（卡片列表 + 简要状态行）
- **SC-005**: 每个额度维度的颜色编码与实际剩余百分比对应，测试覆盖 0%、2%、20%、50%、80%、100% 六个关键值
- **SC-006**: 弹出窗口宽度在 320-380px 范围内，内容区无横向溢出
