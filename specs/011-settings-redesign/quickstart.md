# 快速开始：设置页面重设计

**特性**：011-settings-redesign  
**日期**：2026-03-23

## 前置条件

- 通过 `nvm use` 使用 Node.js 20 LTS
- 已安装 Rust stable toolchain
- 如需验证真实壳层拖拽与窗口高度表现，优先在 macOS Tauri 环境中检查

## 开发工作流

```bash
# 前端 + 宿主壳层
npm run tauri:dev

# 前端测试
npm test

# Rust 回归
cargo test

# 定向壳层与截图检查
npm run test:e2e:tauri
npm run test:e2e:screenshots
```

## 验证清单

### 1. 一屏可见与视觉一致

1. 打开设置页。
2. 确认 7 个设置项在默认窗口高度内完整可见，无需滚动。
3. 确认不存在 `显示 / 通用 / 连接 / 状态` 分区标题。
4. 确认设置卡片圆角、边框、背景和阴影与主面板卡片风格一致。

### 2. 行内控件与即时保存

1. 依次修改托盘摘要规则、菜单栏服务、语言与刷新间隔。
2. 每次操作后确认无需额外确认按钮。
3. 确认 header 显示“保存中”再短暂切换为“已保存”。
4. 人为制造一次保存失败，确认控件值会回滚到最近一次成功保存的状态。

### 3. 开机自启与排序

1. 切换开机自启开关。
2. 确认开关位于行右侧且即时生效。
3. 拖拽“面板顺序”胶囊调整顺序。
4. 返回面板确认服务卡片顺序已同步变化。
5. 在较窄宽度或更多服务场景下，确认胶囊可换行且不会破坏拖拽命中区域。

### 4. 代理自动保存

1. 将网络代理切换为“系统代理”。
2. 确认立即保存，且没有“应用”按钮。
3. 切换为“手动输入”。
4. 确认展开 URL 输入框。
5. 输入无效 URL 并失焦，确认出现行内错误。
6. 输入有效 URL 并失焦，确认自动保存成功。
7. 人为制造一次代理保存失败，确认模式或 URL 回滚到最近一次成功持久化的值。

### 5. 状态区移除

1. 浏览完整设置页。
2. 确认不存在“状态”区块。
3. 确认不再显示 data source、active session 等调试信息。

## 回归重点

- 默认窗口高度下首屏不能被新布局撑破
- 中英文切换下标签与控件不能互相挤压失真
- 代理移除“应用”按钮后仍必须保留真实校验和错误反馈
- 排序失败时必须能回退到已保存顺序
- 即时保存失败时，select / switch / 代理模式都必须回滚到最近一次成功值
- serviceOrder 胶囊在溢出时必须换行而不是挤压出界
- 删除状态区后，设置页不能残留其它调试信息入口
