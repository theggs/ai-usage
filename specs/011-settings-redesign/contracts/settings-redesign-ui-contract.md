# UI 契约：设置页面重设计

**特性**：011-settings-redesign  
**日期**：2026-03-23

本文档定义 011 对设置页的稳定用户可见行为。它覆盖 009/010 中已不再适用的设置页旧契约，尤其是：
- 旧的分组式布局约束
- 代理“显式应用”按钮路径
- 设置页“状态”区展示要求

---

## 1. 布局契约

### 页面结构

- 设置页必须移除分区标题和多层 section 容器。
- 所有正式设置项必须位于一张统一的圆角卡片容器内。
- 各设置项之间必须使用水平分隔线，而不是独立卡片堆叠。
- 每个设置项必须采用“左侧标签，右侧控件”的行内布局。

### 首屏可见性

- 在默认窗口高度下，7 个设置项必须完整可见，无需滚动。
- 当未来设置项数量增长超出首屏高度时，页面可以退化为滚动容器，但不得破坏单卡片行列表结构。

### 视觉一致性

- 设置卡片的圆角、边框、背景、阴影与字体层级必须与主面板服务卡片保持一致。
- 设置页不得再使用蓝紫色、大写、加大字距的 section heading 风格。

---

## 2. 即时保存契约

以下设置项必须在用户完成一次操作后立即保存，无需额外确认按钮：

- `traySummaryMode`
- `menubarService`
- `serviceOrder`
- `language`
- `autostartEnabled`
- `refreshIntervalMinutes`
- `networkProxyMode`
- `networkProxyUrl`

### 保存反馈

- 保存中时，设置页 header 必须显示统一的“保存中”状态。
- 保存成功时，header 必须短暂显示“已保存”，随后恢复空闲态。
- 保存失败时，header 必须显示失败状态，并保留真实错误反馈。
- 对于已持久化驱动的 select、switch、排序与代理模式切换，保存失败时 UI 必须回滚到最近一次成功持久化的值。

---

## 3. 控件契约

### Select 行

- 托盘摘要规则、菜单栏服务、语言、刷新间隔、网络代理模式必须渲染为右对齐 select。
- 刷新间隔只能提供以下预设值：
  - `5 分钟`
  - `10 分钟`
  - `15 分钟`
  - `30 分钟`

### Switch 行

- 开机自启必须在同一行右侧渲染为 toggle switch。
- 点击 switch 后必须立即持久化。

### 排序行

- 面板顺序必须以单行胶囊标签呈现，不得继续使用两张纵向拖拽卡片。
- 双服务及以上时，胶囊必须支持拖拽排序。
- 单服务时，胶囊仍可见，但拖拽必须禁用。

---

## 4. 代理契约

### 模式切换

- 选择 `系统代理` 或 `不使用代理` 时，必须立即保存。
- 选择 `手动输入` 时，必须展开一个 URL 输入框。

### URL 输入

- 手动 URL 输入框只在 `manual` 模式下显示。
- URL 必须沿用现有校验规则，要求完整 URL。
- 校验错误必须以内联方式显示在“网络代理”设置项上下文内。
- 正式设置页中不得再渲染“应用”按钮。
- 手动 URL 校验失败时不得覆盖最近一次已保存的代理值；输入框可以保留当前草稿供用户继续修正。

### Claude Code 联动

- 代理设置持久化后，既有 Claude Code 刷新联动必须继续生效。

---

## 5. 移除行为契约

- 设置页不得再渲染“状态”区块。
- 设置页不得展示 data source、active session 或其它面向开发调试的状态摘要。
- 设置页不得要求用户点击通用“保存设置”按钮。

---

## 6. 失败与回退契约

- 即时保存失败时，UI 必须真实反映未保存成功，而不是停留在乐观态。
- 即时保存失败时，UI 必须回滚到最近一次确认成功的持久化值。
- 排序持久化失败时，顺序必须回退到最近一次确认成功的状态。
- 手动代理 URL 校验失败时，必须阻止保存并显示明确错误。
